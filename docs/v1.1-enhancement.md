# Rilis v1.1 - Administrator Module Enhancement

Dokumen ini berisi catatan perubahan (*changelog*) untuk versi 1.1 yang berfokus pada fitur modul administrator (Manajemen Pengguna dan Manajemen Peran).

## 1. Perubahan Database
Terdapat penyesuaian skema pada tabel `users` untuk memfasilitasi kelengkapan profil pengguna di institusi/organisasi:
- **`username`**: Kolom baru (VARCHAR 50, UNIQUE, NOT NULL). Digunakan sebagai pengenal utama (*identifier*).
- **`nip`**: Kolom baru (VARCHAR 50, NULL). Nomor Induk Pegawai opsional.
- **`phone_number`**: Kolom baru (VARCHAR 20, NULL). Kontak yang bisa dihubungi.

*Catatan: Seed data awal telah diperbarui untuk mengakomodasi kolom `username` secara default (`superadmin`, `financeadmin`, `manager`).*

## 2. Fitur Backend (API)
- **Modul `User` (`/api/v1/users`)**: 
  - Penambahan *endpoint* CRUD lengkap (Create, Read, Update, Delete) untuk pengguna.
  - Implementasi *Bcrypt hashing* untuk keamanan *password* saat membuat atau mengedit kredensial pengguna.
  - Validasi pencegahan penghapusan data pengguna yang telah terhubung ke *history* transaksi.
- **Modul `Role` (`/api/v1/roles`)**: 
  - Penambahan *endpoint* CRUD lengkap untuk mengatur *Role Based Access Control* (RBAC).
  - Validasi pencegahan penghapusan *Role* apabila masih terdapat pengguna aktif yang bernaung di bawah *Role* tersebut.
- **Audit Logging**: 
  - Seluruh *endpoint* Administrator telah diintegrasikan dengan tabel `audit_logs` untuk mencatat aktivitas secara otomatis (CREATE, UPDATE, DELETE).

## 3. Fitur Frontend (UI/UX)
- **Menu Administrator**:
  - Penambahan sub-kategori menu *Administrator* pada komponen Sidebar (`Sidebar.jsx`). Menu ini dilindungi *(protected)* dan hanya akan terlihat oleh pengguna yang memiliki otoritas `users.read` atau `roles.read` (maupun Super Admin dengan hak akses `*`).
- **Halaman User Management**:
  - Tampilan daftar *(table)* seluruh pengguna yang terdaftar di sistem.
  - Form interaktif dengan kolom mandatory `Username`, `Email`, `Password`, serta `Role`.
  - Kolom pendukung opsional: `NIP` dan `Nomor HP`.
  - Sistem pengubahan/pengaturan status aktif pengguna.
- **Halaman Role Management**:
  - Tampilan daftar peran.
  - Form pembuatan peran baru, terintegrasi dengan modul pemilihan Hak Akses (Permissions) bergaya *Checkbox* interaktif (mencakup Modul *Dashboard*, *COA*, *Transactions*, *Approvals*, *Budgets*, *Reports*, dll).
